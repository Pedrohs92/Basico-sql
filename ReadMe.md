# Estudos de SQL para QA

## Sobre o Projeto

Este projeto tem como objetivo adquirir e ampliar conhecimentos em SQL, desenvolvendo uma base sólida para apoiar atividades relacionadas à qualidade de software (QA).

O estudo foi focado na prática de comandos fundamentais para consulta e manipulação de dados, permitindo uma melhor compreensão da estrutura dos bancos de dados e maior autonomia na análise e validação de informações durante os testes.

## Comandos SQL Estudados

| Comando       | Função                                               |
| ------------- | ---------------------------------------------------- |
| `SELECT`      | Consultar dados de uma ou mais tabelas.              |
| `WHERE`       | Filtrar registros com base em condições específicas. |
| `ORDER BY`    | Ordenar os resultados de uma consulta.               |
| `GROUP BY`    | Agrupar registros com características em comum.      |
| `HAVING`      | Filtrar resultados após agrupamentos.                |
| `INSERT INTO` | Inserir novos registros em uma tabela.               |
| `UPDATE`      | Atualizar dados existentes.                          |
| `DELETE`      | Excluir registros de uma tabela.                     |
| `JOIN`        | Relacionar dados entre duas ou mais tabelas.         |
| `COUNT`       | Contar a quantidade de registros.                    |
| `DISTINCT`    | Retornar valores sem duplicidade.                    |
| `LIMIT`       | Restringir a quantidade de registros retornados.     |

## Objetivo

Fortalecer os conhecimentos em banco de dados e SQL para apoiar atividades de testes, análise de dados, validação de informações e investigação de cenários, contribuindo para uma atuação mais completa e eficiente na área de Quality Assurance.
